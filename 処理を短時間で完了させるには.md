# 処理を短時間で完了させるには

CPU の pyannote より**短い時間**で同じ目的（話者付き文字起こし）を達成するための設定です。

---

## 1. 話者分離エンジンは「ローカル（diarize）」にする

- GUI の **「話者分離エンジン」** で **「ローカル（diarize）」** を選んでください。
- **効果**: 話者分離が pyannote（CPU）より**かなり短時間**で終わります。HF トークンも不要です。
- `config.py` の `DIARIZATION_BACKEND` が `"diarize"` なら、GUI を開いたときの初期表示も「ローカル」になります。以前 pyannote を選んでいた場合は、**いちど「ローカル（diarize）」に切り替えて**実行すると短時間で完了します。

---

## 2. （任意）Whisper モデルを小さくする

- 文字起こしの**速度**を優先する場合は、**Whisper モデル**を **small** や **base** にすると、さらに全体時間が短くなります。
- **精度**を優先する場合は **medium** や **large-v3** のままにしてください。

| モデル   | 目安（速度） | 目安（精度） |
|----------|--------------|--------------|
| tiny     | 最速         | 低           |
| base     | 速い         | やや低       |
| small    | 標準         | 標準（既定） |
| medium   | やや遅い     | 高           |
| large-v3 | 遅い         | 最高         |

---

## 3. まとめ（最短時間の組み合わせ）

| 項目             | 推奨設定                          |
|------------------|-----------------------------------|
| 話者分離エンジン | **ローカル（diarize）**           |
| Whisper モデル   | **small**（既定）または **base**  |
| 話者分離         | 自動 または 常に音声話者分離      |

この組み合わせで、**CPU で pyannote を使うより短い時間**で、話者付きの文字起こし結果を得られます。設定は GUI で保存されるので、次回以降もそのまま使えます。
